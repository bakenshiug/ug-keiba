第 二 章

機 材 紹 介

Fifine マイク（推定 K688 / AM8）

底面に XLR + USB-C + 3.5mm ヘッドホン端子 を備えるダイナミックマイク。
Shure SM7B のクローン的存在で、コスパ最強。

項目仕様・特徴
形式ダイナミックマイク
接続USB-C 直挿し（Mac対応）／ XLR（IF経由）
サンプルレート48 kHz / 24bit
指向性カーディオイド（単一指向性）
ヘッドホン端子3.5mm（ゼロレイテンシ・モニタリング可）
強み環境音を拾いにくい・近接効果で太い声

💡 神宮ポイント

ダイナミックマイクは、コンデンサーマイクと違って部屋の反響や生活音をほぼ拾わない。 エアコンの音、家族の足音、PCファン音などが配信に乗らない＝神宮の静寂感を保てる神アイテム。

Mac（既存）

OBS は Mac 公式版で問題なし。M1 以降のチップなら配信中の動画エンコード負荷もラクラク。

OBS Studio（無料・公式）

https://obsproject.com/ よりダウンロード。 ライブ配信・録画両方できるオープンソース。

任意：Webカメラ（顔出し用）

サングラス＋帽子の神主ニックスタイルで顔出し配信予定。 Mac内蔵カメラでもOK、外付けなら Logicool C920 等で十分。

第 三 章

マイクセットアップ

Step 1: 物理接続

  1. USB-C ケーブルでMacに直接接続
  2. マイクの正面ロゴ「fifine」が自分側を向くよう設置
  3. 任意：3.5mm ヘッドホンをマイク底面に挿す（モニタリング用）

Step 2: Mac サウンド設定

# システム設定 → サウンド → 入力 入力デバイス: Fifine Microphone を選択 入力レベル: メーターが中盤に来るよう調整（喋ってチェック） # システム設定 → サウンド → 出力（ヘッドホン使用時） 出力デバイス: Fifine Microphone（マイク経由のヘッドホンモニタリング）

Step 3: マイクポジション

  • 顔から拳1個分（約 10cm）離す
  • 真上向きにせず、口元にやや向ける
  • 「ポ」「パ」「バ」の破裂音は口を斜めにズラして発音
  • フォームウィンドスクリーン（黒いスポンジ）が付属＝ポップガード代わり

⚠ よくある失敗

  • マイクが遠い → 声が薄く、エコー混入
  • マイクの真上で話す → 高音抜けが悪い
  • USB-C 半挿し → 認識不安定 ※ 奥までしっかり挿す

第 四 章

OBS インストール＆基本設定

Step 1: ダウンロード

https://obsproject.com/ja にアクセス → macOS 版をダウンロード → インストール

Step 2: 自動構成ウィザード

初回起動時に自動構成ウィザードが表示される。以下を選択：

  • 使用目的：配信を最適化する
  • 映像設定：1920×1080 / 30fps（ツイキャス無料枠は720pでも可、まず1080pで設定）
  • サービス：「カスタム…」を選択（ツイキャス用RTMP後で設定）

Step 3: 出力設定

# 設定 → 出力 → 配信 エンコーダー: Apple VT H264 ハードウェアエンコーダ ビットレート: 3500 Kbps（720pなら2500、1080pなら3500-4500） キーフレーム間隔: 2 # 設定 → 音声 サンプルレート: 48 kHz チャンネル: ステレオ マイク音声: Fifine Microphone

第 六 章

OBS 音声フィルター（神宮推奨）

OBS の「音声入力キャプチャ」を右クリック →「フィルタ」を開き、以下4種を順番に追加。

① ノイズ抑制

方法: RNNoise (高品質)

エアコン音・PCファン音をAIで除去。神宮の静寂感UP。

② ノイズゲート

クローズしきい値: -45 dB オープンしきい値: -35 dB アタック時間: 25 ms リリース時間: 150 ms

喋っていない時は完全無音。生活音漏れ防止。

③ コンプレッサー

比率: 4 : 1 しきい値: -18 dB アタック: 6 ms リリース: 60 ms 出力ゲイン: +3 dB

大声/小声の音量差を均す。神主の落ち着いた語り口を均一化。

④ ゲイン（必要に応じて）

+3 ～ +6 dB

全体的に小さければ持ち上げ。OBSメーターが緑〜黄ゾーンに収まればOK。 赤に振り切らないよう注意（音割れ）。

💡 配信前テスト

OBSの「録画」ボタンで30秒テスト録画 → 再生して以下を確認：

  • 声量適正か
  • 無音時にホワイトノイズなしか
  • 環境音漏れていないか

OBSでSUNO音源を流す技術

SUNO音源を MP3 ダウンロード → Mac内のフォルダ整理 → OBSで再生する流れ。 方法AとB の2種、神宮的にはBがオススメ。

音源フォルダ整理（推奨）

~/Music/UG-Jingu-BGM/ ├── 01_morning_shrine/ # 朝の神事 30曲 ├── 02_afternoon/ # 午後 30曲 ├── 03_main/ # メイン神事 10曲 ├── 04_evening/ # 夕の神事 30曲 └── 05_jingles/ # ジングル 10種

【方法A】OBS メディアソース（シンプル）

  1. OBS → ソース → 「+」→「メディアソース」追加
  2. 設定:
    名前: 朝の神事BGM ローカルファイル: ~/Music/UG-Jingu-BGM/01_morning_shrine/track01.mp3 ループ: ✅ ON 音声出力をモニター: なし（配信に乗せる）
  3. 音声ミキサーで音量を -15dB に下げる

メリット: シンプル・即動く
デメリット: 1ファイル1ソース、複数曲ローテは手動

【方法B】VLCビデオソース（プレイリスト・神宮推奨）

30曲を勝手にシャッフル＆ループ再生してくれる神機能。

  1. VLC Media Player インストール（無料）
    https://www.videolan.org/vlc/
  2. OBS → ソース → 「+」→「VLCビデオソース」追加
  3. 設定:
    名前: 朝の神事プレイリスト プレイリスト: ~/Music/UG-Jingu-BGM/01_morning_shrine/ （フォルダ全体追加） ループ: ✅ シャッフル: ✅（飽き防止）

メリット: 30曲を勝手にシャッフル再生・飽きさせない
デメリット: VLCインストール要

視聴者には「マイクの声がメイン、BGMはふんわり背景」というプロのバランスに。

⚠ よくある失敗

    → 必ず「なし」または「出力のみ」
  • VLCインストール後、OBSでVLCソースが選択肢に出ない
    → OBS再起動で解決
